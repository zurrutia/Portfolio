Edge Server Backup at a Bandwidth-Constrained Remote Site
Edge Compute Remote Site Stratus ztC Edge Tosibox VPN OVF / VHD

The Problem

A 120GB historian VM running on a Stratus ztC Edge server at a remote industrial site needed a verified backup. The only connectivity was a Tosibox VPN over a cellular hotspot — effectively a few Mbps of usable bandwidth. Any wireless transfer would have taken days and had a high probability of failure mid-stream.

Constraints

The Stratus platform does not support saving VM exports to its own local filesystem. A CIFS network share is required as the export destination. Routing that share over the Tosibox link would have placed the bandwidth bottleneck directly in the backup path during the export job — making things worse, not better.

Decision Process

Evaluated three paths: a local LAN PC as a CIFS target (fast over LAN, but still requires physical retrieval or secondary transfer), a remote Tosibox-connected PC as the CIFS target (ruled out — worse than doing nothing, as it saturates the constrained link during backup), and USB export directly from the Stratus console to an external drive (viable, avoids the network entirely, requires physical presence at the site).

USB export was selected as the lowest-risk, most reliable path. Documented the approach, risks, and workflow for the operations team and management before any work at the site began.

Forward-Deployed Relevance This is exactly the problem topology that comes up in field-deployed environments: edge hardware, degraded or constrained connectivity, and a need to make sound decisions about data integrity without a datacenter safety net. The ability to reason through it clearly and communicate the tradeoffs to non-technical stakeholders was as important as the technical answer.

On-Prem Active Directory to Entra ID Migration
Identity Architecture Cloud Migration Entra ID Intune Cloud PKI Conditional Access

Scope

Full-scope migration of Agilyx and Cyclyx identity infrastructure from on-premises Active Directory to Microsoft Entra ID. Running as primary technical lead and sole internal IT owner. No large supporting team.

What This Actually Involves

Hybrid identity architecture design using Entra Connect. Conditional Access policy design from scratch, replacing legacy GPO-based controls. Cloud PKI deployment to replace on-prem certificate authority. Device management migration to Intune. RADIUS re-platforming to support cloud-native Wi-Fi and VPN authentication. Security group inventory and cloud-readiness classification across the entire AD environment.

Project Management Approach

Built the full project structure in Asana across ten workstreams. Developed a PERT-based effort estimation model with optimistic / most likely / pessimistic ranges and statistical confidence intervals to give leadership a defensible timeline. Created both an engineering-facing dependency diagram in PlantUML and a leadership-facing capability matrix for stakeholder communication.

Why This Matters Here Identity is foundational to any secure kit deployment. The ability to design IAM architecture, not just administer it, and to own the documentation and stakeholder communication alongside the technical work is what separates an operator from an engineer.

Azure Cost Governance and Decommission Architecture
Cloud Architecture Azure Blob Archive SQL Elastic Pools Legal Retention

Context

Conducted a structured Azure cost review for a decommissioning scenario. Evaluated managed disks, SQL elastic pools and databases, Syntex document processors, and restore point collections across the environment.

Approach

Mapped each resource against legal retention requirements before recommending any deletion. Designed a tiered archival approach using Blob Storage Archive tier as the low-cost long-term hold before permanent removal. Produced a prioritized decommission plan with cost impact and risk flags for each component.

The Takeaway Cloud cost work that ignores legal and compliance requirements creates bigger problems than it solves. The value here was in asking the right questions before touching anything, not just running the cost analysis.

Multi-Site Infrastructure Management at Scale
Multi-Site Hands-On Hardware Windows Server AD / DNS / DHCP SCCM 4,000+ Endpoints

Environment

IT support and infrastructure management across eight school buildings in a hub-and-spoke network topology. Roughly 4,000 Windows endpoints, ranging from classroom workstations to administrative machines and shared lab devices. Responsible for physical provisioning, imaging, and deployment alongside ongoing support.

Infrastructure Ownership

Maintained Windows Server infrastructure across all sites: Active Directory, DNS, DHCP, file and print services. Standardized endpoint builds district-wide using SCCM and Group Policy, which reduced imaging time and eliminated recurring configuration drift between sites.

Automation

Wrote a PowerShell onboarding script to automate account creation for new staff. Cut provisioning time by roughly 40% and eliminated a category of recurring errors that came from manual steps being missed during high-volume onboarding periods at the start of each school year.

Relevant to Forward-Deployed Environments Managing infrastructure across eight physically separate sites with limited staff is operationally similar to supporting distributed kit deployments. You learn to build systems that work without someone standing next to the hardware, to document clearly enough that someone else can follow it, and to treat physical access as a resource to be rationed.
